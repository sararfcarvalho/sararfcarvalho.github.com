Case Study

UI UX Design

Software

Context

On all car brand websites we can find car configurators. Most of the time it helps in the buying decision. But most of them are complex or with many steps, the user ends up tired or frustrated.

The configurator I developed follows a existing style guide (colors, typography, navigation) and was thought to be simple and accessible.

Personas


Manuel

52 yo, Teacher, Braga

Manuel has already had unpleasant surprises when buying used cars, it's time to buy a new one.

When has some free time Manuel likes to surf the web to study his future purchase. As it is a big investment he has to think well and think about all the pros and cons.

User Flow


Final Designs


The screen was divided in two, the user can edit the options and at the same time see the updates in the car.

The first dropdown is always open, as the user selects the options the next dropdown will automatically open. The dropdowns are closed in case of the user only wants to change one of the options, this way he will see all the options at once and select what he wants.

The user can go to the resume in two ways, he has the CTA button on top right and the last menu. The total value is always in the same place, this way the user doesn't get confused.

The garbage button gives the user the ability to reset, but only after a message appears asking if he is sure what he is doing.

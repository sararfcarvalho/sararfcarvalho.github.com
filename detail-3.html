Context

For this academic project the goal was to develop an online magazine with an online shop, the most difficult task was to merge the two concepts.

Sprint Reader is an online magazine covering culture, art, music, cinema, technology, etc. This brand also sells Activewear and Running products on its website.

It has in mind a millenial generation that lives permanently in a sprint. Me and my teammate focused on delivering content to people always on the go.

Based on the interviews we set up two personas.

Frustrations VS Solutions

Some frustrations from our personas lead us to different solutions. These insights allowed us to think about different features.


  • - What if I don't have connection to the Internet? Give the user the possibility to save articles.

  • - I feel like I don't have time to be informed of everything. The user can see articles stories and then decide if he wants to read it.

  • - I don't like reading too long articles. Podcasts.

  • - I like to give feedback. The user can give feedback in the detail product and in the articles page.

  • - I feel like everything is very standard these days. Product customization.

Wireframes & User Testing

After studying the user flow and some sketches we started with the wireframes, then with that material we made the user tests. The tests were made with people from the targeted age, from 20 to 34 years old.

With the user tests we wanted to understand how the user would navigate, how he would add a item to the cart, and if the customization feature was easy to understand and perform.

The user tests were moderated, and we realized that the user needed more control on the cart, like edit colors or sizes. The user also couldn't differentiate the common product of the custom on. In the checkout phase he didn't wanted to login or register but have the chance to buy in a simpler way.

These insights were very important, we have made several changes based on this feedback.

Final Designs

We started designing the final screens in Sketch. First we designed a major part of the UI elements together, defining the look and feel that both wanted for the website.

The biggest problem was that at the time we didn't have a collaborative platform, so we had to divide the screens by each one and communicate constantly, so the screens would follow the same patterns.
